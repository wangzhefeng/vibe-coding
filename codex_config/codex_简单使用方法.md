# Codex 简单使用方法

## 1. 准备环境
- 安装并登录 Codex/CLI 工具。
- 进入你的项目目录：

```bash
cd /path/to/your/project
```

## 2. 启动 Codex
在终端中启动 Codex 会话（根据你的安装方式使用对应命令）：

```bash
codex
```

## 3. 常见使用方式

### 方式一：直接提需求
你可以直接描述目标，例如：
- “帮我创建一个 Python Flask 项目骨架”
- “为这个函数补充单元测试”
- “解释这段报错并修复”

### 方式二：让 Codex 修改文件
- 指定文件路径和修改目标。
- 要求输出补丁或直接写入文件。

示例：
- “请把 `src/app.py` 的日志改为结构化输出。”

### 方式三：执行与验证
让 Codex 帮你执行检查命令，例如：
- 运行测试
- 运行 lint
- 构建项目

## 4. 提示词建议
- 说清楚**目标**：你要实现什么。
- 提供**上下文**：文件位置、技术栈、报错信息。
- 说明**约束**：是否允许新增依赖、代码风格要求。
- 约定**输出格式**：例如“请用 Markdown 列步骤”。

## 5. 一个最小示例

```text
请在当前项目中新增一个 README，包含：
1) 项目简介
2) 安装步骤
3) 启动命令
输出为 Markdown。
```

## 6. 常见注意事项
- 先小步迭代：一次只做一个明确任务。
- 先看 diff 再提交。
- 对关键改动执行测试，确保功能正确。
- 涉及密钥或隐私信息时不要直接粘贴明文。
